# Portfolio Presentation Notes

## Suggested GitHub description

> Compact Mauritian beauty storefront built with Next.js, TypeScript, React Three Fiber, accessible motion and local-first commerce flows.

## Suggested repository topics

`nextjs` `typescript` `react-three-fiber` `threejs` `ecommerce` `mauritius` `responsive-design` `accessibility` `portfolio`

## What this repository demonstrates

- selective WebGL integration inside a commerce interface;
- visual direction and component design;
- React/Next.js route organisation;
- typed product modelling;
- stateful cart interactions;
- compact information architecture;
- responsive horizontal product/category rails;
- progressive enhancement and reduced-motion support;
- honest technical boundaries.

## Recommended pinned-repository order

Pin this repository first once the GitHub source and live deployment are available. It is the strongest customer-facing visual project in the five-repository portfolio plan.

## Before making the repository public

- set the final deployment URL in `NEXT_PUBLIC_SITE_URL`;
- verify the public-safe WhatsApp share flow;
- run the quality checks in `VALIDATION.md`;
- confirm desktop and mobile screenshots;
- create a `v2.1.0` GitHub release;
- add the live Vercel URL to the repository About section.

## Application phrasing

> I built Moris Beauty as an original Mauritius-oriented portfolio concept to demonstrate a mobile-first storefront, a selectively loaded WebGL product stage, searchable catalogue, product pages, cart state, local fulfilment choices and WhatsApp-assisted shopping. The homepage was intentionally compressed to reduce unnecessary scrolling, and it includes reduced-motion and WebGL failure fallbacks.

Do not describe it as:

- a production marketplace;
- a live payment integration;
- a Simple.mu redesign;
- client work;
- a deployed inventory system;
- a fully modelled production 3D catalogue.
